# Caption Review Format

이 문서는 생성 결과를 빠르게 검수하고, 다음 튜닝의 근거를 남기기 위한 포맷이다.

목표
- "뭔가 어색함" 같은 모호한 피드백을 줄인다.
- 좋은 결과와 버릴 결과를 같은 기준으로 분류한다.
- 프롬프트 수정이 아니라 데이터 기반 튜닝으로 넘어갈 수 있게 샘플을 축적한다.

운영 원칙
- 한 샘플마다 30초 안에 판단한다.
- 문장 전체를 다시 고치지 말고, `좋음 / 수정 필요 / 폐기`만 먼저 결정한다.
- 이유는 아래 고정 태그 중 1~3개만 고른다.
- 코멘트는 한 줄만 쓴다.

## 판정 등급

### 1. 좋음
- 바로 인스타에 올려도 큰 위화감이 없다.
- 톤이 선택값과 실제로 다르게 느껴지지 않는다.
- 입력 정보가 자연스럽게 녹아 있다.

### 2. 수정 필요
- 살릴 수는 있지만, 어색한 부분이 1~2개 있다.
- 톤은 대체로 맞지만 문장 리듬이나 마무리가 아쉽다.

### 3. 폐기
- 사람이 쓴 느낌이 약하다.
- 톤이 어긋났거나, 광고문/공지문/오글거리는 문장으로 보인다.
- 입력 내용을 거의 옮겨 적었거나 의미가 이상하다.

## 고정 태그

아래 태그만 사용한다.

### 톤 관련
- `tone_emotional_flat`
  - 감성 톤인데 설명문처럼 밋밋함
- `tone_emotional_over`
  - 감성 톤이 과하게 오글거리거나 감상문 같음
- `tone_casual_formal`
  - 캐주얼인데 공지문/안내문 같음
- `tone_casual_stiff`
  - 캐주얼인데 말이 너무 굳어 있음
- `tone_professional_dry`
  - 전문 톤인데 너무 딱딱하고 관공서 문장 같음
- `tone_professional_weak`
  - 전문 톤인데 전문적으로 느껴지지 않음

### 문장 관련
- `ending_monotone`
  - 문장 끝이 전부 `~요`, `~네요`, `~습니다`처럼 반복됨
- `template_phrase`
  - 템플릿 냄새가 나는 익숙한 표현이 나옴
- `ai_like`
  - 사람이 쓰지 않을 말투나 어색한 조합이 나옴
- `broken_text`
  - 조사, 단어, 문장이 깨져 있음
- `length_short`
  - 너무 짧아서 성의 없어 보임
- `length_long`
  - 너무 길어서 읽기 무거움

### 내용 관련
- `context_copied`
  - 오늘 상황 입력 문장을 거의 그대로 옮김
- `context_missing`
  - 입력한 오늘 상황이 결과에 거의 반영되지 않음
- `detail_weak`
  - 현장감 있는 디테일이 부족함
- `owner_voice_weak`
  - 사장님이 직접 쓴 느낌이 약함
- `reporting_voice`
  - 사장님이 직접 쓰는 글이 아니라 직원이나 외부인이 대신 보고하는 말투처럼 들림
- `promo_smell`
  - 홍보문/광고문 냄새가 남

## 샘플 기록 포맷

```md
## Sample 001
- industry: cafe
- tone: EMOTIONAL
- judgment: 수정 필요
- tags: ending_monotone, template_phrase
- input:
  - content: 딸기라떼 새로 만들었고 당도를 낮췄다
  - weather: 비 오는 날이라 따뜻한 음료 주문이 많음
  - reaction: 시음 손님이 덜 달아서 좋다고 함
- output:
  - 비 오는 날씨에 딸기라떼를 새롭게 조정했어요. 당도를 낮추니 부드럽고 상큼한 맛이 더 살아났네요. 시음해본 손님이 덜 달아서 좋다고 하셨던 게 기억에 남아요. 따뜻한 음료 주문이 많아지니 뭔가 흐뭇하네요.
- note:
  - 감성 톤은 맞는데 마지막 문장이 템플릿 같고 문장 끝 리듬이 비슷함.
```

## 권장 수집량

처음에는 아래 정도면 충분하다.

- 업종 3개: `cafe`, `restaurant`, `salon`
- 톤 3개: `EMOTIONAL`, `CASUAL`, `PROFESSIONAL`
- 업종별 10개씩
- 총 30개

이 30개만 모여도 다음 튜닝 우선순위가 보인다.

## 검수 파일 작성 규칙

- 파일명 예시: `/Users/tuesdaymorning/Devguru/sajangnim_insta/data/caption_review_2026-03-01.md`
- 샘플은 번호를 붙여 누적한다.
- 한 번에 10개 이상 몰아서 보지 않는다.
- 각 샘플은 반드시 `judgment`와 `tags`를 남긴다.

## 다음 튜닝 기준

아래처럼 5개 이상 반복되면 튜닝 대상으로 본다.

- `ending_monotone` 5건 이상
- `tone_professional_dry` 5건 이상
- `tone_casual_formal` 5건 이상
- `tone_emotional_over` 5건 이상
- `reporting_voice` 3건 이상
- `context_copied` 3건 이상

## 최소 운영 절차

1. 사이트에서 샘플 10개 생성
2. 이 포맷으로 바로 판정
3. `폐기`만 따로 세기
4. 가장 많이 나온 태그 2개만 다음 수정 대상으로 선택
5. 수정 후 다시 10개만 재검수

핵심은 한 번에 다 고치려 하지 않는 것이다.
